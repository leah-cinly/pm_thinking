# Question Bank

Use these as seeds. Adapt wording to the product, user segment, and source facts.

## Mature Product System

- Why does this product own the user's starting point for the scenario?
- Which part of the product is infrastructure rather than feature?
- What behavior would users keep even if the UI changed completely?
- Which metric could the team optimize that would quietly damage trust?
- What is the product's most important default setting?
- Where does the product reduce user decision cost?
- Which user habit is the product protecting at all costs?
- What would break if this product removed its most familiar friction?

## New Product or Feature

- What platform shift made this product possible now?
- Is the feature a workflow, a wedge, or a full product?
- What job-to-be-done is being pulled out of an older tool?
- What distribution advantage did the product use before competitors copied the feature?
- Which early users are most forgiving, and why?
- What would make the product habit-forming rather than merely impressive?
- What makes the product trustworthy enough for users to delegate work?
- Which failure mode would stop mainstream adoption?

## Creative Design Detail

- What does the empty state teach users to do?
- Which friction is intentionally kept because it protects quality or trust?
- Where does the product turn uncertainty into a visible next step?
- What is the smallest cue that changes user confidence?
- How does the product help users recover from mistakes?
- Which microcopy changes the user's perceived effort?
- Where does the product ask for commitment, and is the timing right?
- What does the product hide so the user can make a simpler decision?

## Strategy and Market

- Which competitor is the product choosing not to fight?
- What moat compounds with usage?
- What is the bundling logic?
- Which user segment is the beachhead and which is the expansion path?
- What is the pricing unit, and what behavior does it encourage?
- What would a stronger incumbent do in response?
- Which scarce resource is the strategy trying to own: attention, data, distribution, trust, workflow, or budget?
- What second-order effect appears after competitors copy the feature?

## Product Performance

- Why was this project worth doing at this stage?
- Which team/company OKR does it support?
- What leading indicator should move first?
- What user behavior should change within one session?
- What did the PM contribute beyond completing tasks?
- Which tradeoff was made visible to stakeholders?

## Market Demand

- Who is the core user, and who is only an adjacent user?
- What real user pain is being solved: time, money, uncertainty, status, risk, creativity, or coordination?
- What substitute is the user currently using?
- Why would users solve this on this platform rather than elsewhere?
- What competitor behavior proves the demand exists?
- What user segment would refuse to switch, and why?

## Product Value

- What value is created for the user?
- What value is created for the platform?
- What value is created for commercial growth?
- Does usage create a durable advantage such as data, workflow lock-in, user mindshare, network effect, or ecosystem dependency?
- Which value is strongest now, and which is only a future promise?

## Personal Work Reflection

- How did you find the problem or scenario?
- What was your contribution beyond execution?
- Which indicator connects your work to team OKR?
- What did users do differently after the change?
- What tradeoff did you make, and why was it the right tradeoff?
- What would you do differently with half the resources?
- What evidence would make you admit the direction was wrong?

## Daily Output Pattern

```markdown
今日主问题：
如果你是 [product/company] 的 PM，面对 [scenario], 你会如何判断 [tradeoff]？

隐藏假设：
The product is betting that [user/market behavior] will happen because [reason].

10 分钟练习：
写出 1 个证据、1 个反证、1 个实验、1 个你会放弃的方案。
```

## Five-Question Daily Set

Use this structure when the user asks for a daily push:

1. `成熟产品`: one question about a mature product's system logic.
2. `新成功产品`: one question about timing, wedge, or adoption.
3. `创意细节`: one question about a specific design detail.
4. `战略布局`: one question about moat, ecosystem, distribution, or commercialization.
5. `个人复盘`: one question that maps to the user's own work, OKR, contribution, and evidence.
