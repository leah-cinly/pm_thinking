# Notion Daily Push

Use this reference when publishing daily product-thinking practice to Notion `产品思维训练营`.

## Destination

- Target name: `产品思维训练营`.
- Prefer an existing Notion database/page with this exact name.
- If multiple matches exist, choose the most recently edited training/workspace page or ask the user.
- If Notion tools are unavailable, provide the complete Markdown page content and explicitly say the push was not completed.

## Topic Mix Rules

- Include at least one mature product: WeChat, YouTube, TikTok/Douyin, Amazon, Notion, Figma, GitHub, ChatGPT, Claude, Spotify, Duolingo, etc.
- Include at least one recent successful product/feature based on current research.
- Include one micro-design detail: onboarding, default, pricing, empty state, prompt design, permission request, error recovery, trust cue, notification, collaboration, or retention loop.
- Include one strategy question: ecosystem, distribution, moat, bundling, commercialization, platform shift, internationalization, data advantage, regulation, or competitor response.
- Include one personal reflection question that connects the user's own work to product performance, market demand, and product value.

## Notion Page Quality Bar

- Questions should force judgment, tradeoff, and evidence, not recall.
- Product judgment must be concrete enough that the user can write a PM-style answer.
- Use source links with dates when available.
- Separate facts from inference.
- Do not cite inaccessible social posts as evidence.
